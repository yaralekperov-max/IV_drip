# С чего начать — первое задание для Claude Code

Этот файл — подсказка тебе (владельцу) о том, что попросить Claude Code сделать первым.

## Шаг 0 — дай Claude Code осмотреться

Открой проект в Claude Code и первым сообщением попроси:

> Прочитай CLAUDE.md, docs/tz.md, docs/architecture.md и docs/decisions.md.
> Посмотри прототипы в prototypes/. Опиши, как ты понял проект, и предложи план разработки по шагам.

Так он загрузит весь контекст и предложит план. Проверь, что он всё понял правильно.

## Шаг 1 — каркас проекта

> Создай каркас Next.js проекта с TypeScript и Tailwind CSS. Настрой структуру папок
> по CLAUDE.md (app/, lib/modules/, lib/integrations/, components/). Подключи Supabase.
> Перенеси дизайн-систему из прототипов (цвета, шрифты) в Tailwind-конфиг.

## Шаг 2 — лендинг

> Собери лендинг как React-компоненты на основе prototypes/landing.html.
> Сохрани визуал и продуктовые модалки. Форму заявки пока сделай заглушкой
> (позже подключим amoCRM).

## Шаг 3 — авторизация и ЛК

> Настрой вход по SMS через Supabase Auth. Собери личный кабинет по
> prototypes/client-portal.html — все разделы и состояния (новый/активный клиент).
> Данные пока моковые.

## Шаг 4 — админка

> Собери админ-панель по prototypes/admin-panel.html. Особое внимание — экран
> проверки анализов врачом (наш уникальный модуль).

## Шаг 5 — база данных

> Спроектируй схему БД в Supabase: клиенты, визиты, биомаркеры, платежи.
> Учти изоляцию медданных (152-ФЗ).

## Шаг 6 — интеграции (по готовности)

> Подключи amoCRM (создание лида из формы заявки) и ЮKassa (оплата визита).

---

## Важные напоминания для Claude Code

- Прототипы в `prototypes/` — референс, не копируй HTML напрямую, пересобирай в компоненты.
- Медданные не отправляй во внешние сервисы без согласия (152-ФЗ) — см. decisions.md.
- Спрашивай перед решениями, затрагивающими медданные или юридические аспекты.
- Не коммить секреты (.env уже в .gitignore).

## Как заливать изменения в GitHub

Claude Code сам умеет работать с git. Проси его коммитить осмысленными порциями:

> Закоммить текущие изменения с понятным сообщением и запушь в main.
