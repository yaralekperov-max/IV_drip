# VENA — выездная IV-терапия

Премиальный сервис выездных капельниц в Москве. Веб-приложение: лендинг, личный кабинет клиента и админ-панель.

## Что это

Клиент заказывает капельницу через сайт или личный кабинет — бригада (медсестра под врачебным контролем) приезжает к нему домой, в офис или между встречами. Ключевая особенность — личный кабинет с динамикой анализов «было → стало» под контролем врача.

## Стек

- **Next.js** (React, TypeScript) — фронтенд + API
- **Tailwind CSS** — стили
- **Supabase** — БД, авторизация, хранение файлов
- **ЮKassa** — платежи
- **amoCRM** — CRM для операционки (интеграция)

## Структура

- `app/` — Next.js приложение (лендинг, ЛК, админка, API)
- `lib/modules/` — бизнес-логика по модулям (клиенты, заявки, биомаркеры, платежи)
- `lib/integrations/` — внешние интеграции (amoCRM, лаборатории)
- `components/` — UI-компоненты
- `prototypes/` — исходные HTML-макеты (референс дизайна и логики)
- `docs/` — техзадание, архитектура, зафиксированные решения

## Начало работы

Проект в стадии разработки. Основной контекст — в `CLAUDE.md`.
Документация — в `docs/`.

## Прототипы

В `prototypes/` — кликабельные HTML-макеты всех частей:
- `landing.html` — лендинг
- `client-portal.html` — личный кабинет клиента (полный, все экраны и состояния)
- `admin-panel.html` — админ-панель
- `biomarker-parser.html` — парсер анализов (ИИ)

Открываются в браузере. Это референс для реализации, не финальный код.

## Важно

Проект работает с медицинскими персональными данными (152-ФЗ). Обработка анализов требует согласия клиента и защищённого хранения. См. `docs/decisions.md`.
